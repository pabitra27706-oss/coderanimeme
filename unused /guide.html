Owner Guide

How to update and maintain your website

This page is your practical guide. It explains exactly how to add new projects, how to add new videos with copyable code, what files you must update, and what mistakes to avoid.

1. Your normal workflow

You will mostly work in these files and folders:

assets/data/index.json assets/data/projects/ assets/data/videos/ assets/code/ assets/images/projects/
The website engine is already built. Most of the time you do not need to edit JavaScript or CSS. You only add content files.

2. How to add a new project

Step-by-step project workflow

  1. Create a separate GitHub repository for the project
  2. Create a thumbnail image in assets/images/projects/
  3. Create a new project JSON file in assets/data/projects/
  4. Add the project entry to assets/data/index.json
  5. Test projects.html
  6. Test project.html?id=your-project-id

Project JSON template

{ "id": "project-002", "title": "Project Title", "description": "Short description.", "longDescription": "Long project explanation.", "category": "python", "tags": ["python", "turtle"], "status": "completed", "thumbnail": "assets/images/projects/project-image.png", "dateCreated": "2025-01-10", "repoUrl": "https://github.com/pabitra27706-oss/project-repo", "liveUrl": null, "relatedVideo": "video-002", "whatILearned": [ "Lesson one", "Lesson two" ] }

3. How to add a new video with copyable code

Step-by-step video workflow

  1. Create a new video JSON file in assets/data/videos/
  2. Create a code folder like assets/code/video-002/
  3. Add the real code files inside that folder
  4. Add the video entry to assets/data/index.json
  5. Test videos.html
  6. Test video.html?id=video-002
  7. Check that copy buttons work

Video JSON template

{ "id": "video-002", "title": "Video Title", "description": "Short description.", "longDescription": "Long explanation.", "category": "tutorial", "tags": ["python", "tutorial"], "thumbnail": "assets/images/projects/project-image.png", "youtubeUrl": "https://youtube.com/watch?v=YOUR_LINK", "youtubeId": null, "datePublished": "2025-01-10", "duration": "08:00", "relatedProject": "project-002", "sourceRepo": "https://github.com/pabitra27706-oss/project-repo", "websiteLink": null, "files": [ { "name": "main.py", "path": "assets/code/video-002/main.py", "language": "python" } ] }

4. How the code copy system works

What you need

For each tutorial video:

  • A video JSON file
  • A folder in assets/code/
  • One or more real source files inside it
  • The correct file list inside the video JSON

Example file structure

assets/code/video-002/ ├── main.py ├── helper.py └── README.txt

Then list those inside the video JSON files array.

5. The one file you must never forget

assets/data/index.json

This is the master list for all your project and video files.

If you create a new project or video but forget to update this file, the site will not show the content.

{ "projects": [ { "id": "project-002", "file": "assets/data/projects/project-002.json", "featured": true } ], "videos": [ { "id": "video-002", "file": "assets/data/videos/video-002.json", "featured": true } ] }

6. What you always need to update

For every new project

  • Create new project JSON file
  • Add project to index.json
  • Add thumbnail image
  • Set GitHub repo URL
  • Set related video if available

For every new video

  • Create new video JSON file
  • Add video to index.json
  • Create code folder
  • Add source files
  • Set YouTube URL
  • Set related project if available

7. Rules you should always follow

  • Every project must have a unique id
  • Every video must have a unique id
  • Every new project/video must be added to index.json
  • Project and video IDs should match related links properly
  • File paths must be exact
  • JSON must be valid with correct quotes and commas
  • Project repos should be separate GitHub repositories
  • Video code should live in assets/code/video-XXX/

8. Common mistakes to avoid

Content mistakes

  • Forgetting index.json
  • Using wrong image path
  • Using wrong code file path
  • Wrong project/video id

JSON mistakes

  • Missing comma
  • Using single quotes instead of double quotes
  • Trailing comma in JSON
  • Broken array brackets

9. Testing checklist

  • Does the new card appear on the list page?
  • Does the detail page open with the correct id?
  • Does the thumbnail load?
  • Does the GitHub repo button open correctly?
  • Does the related project/video link work?
  • For videos: do the code files load?
  • For videos: does the copy button work?
  • For videos: does the copy all button work?

10. Short summary

For projects: create project JSON + add to index.json + add thumbnail + add GitHub repo.

For videos: create video JSON + create code folder + add code files + add to index.json.

If you only remember one thing, remember this: new content = JSON file + index.json update + correct file paths.
