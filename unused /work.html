File Work Reference

What each file does in your website

This page explains the job of every important file in your CoderAnimeMe website project. Use this as your internal reference whenever you forget what a file is for.

1. Main HTML pages

index.html
Home page

Shows the main introduction, featured projects, featured videos, skills, roadmap, and call-to-action sections.

projects.html
Projects list page

Shows all project cards loaded from project JSON files.

videos.html
Videos list page

Shows all video cards loaded from video JSON files.

about.html
About page

Explains who CoderAnimeMe is, the learning journey, and the creator direction.

contact.html
Contact page

Shows links to GitHub, YouTube, and future contact methods.

project.html
Dynamic project detail page

Loads one project using a URL like project.html?id=project-001.

video.html
Dynamic video detail page

Loads one video using a URL like video.html?id=video-001 and shows copyable source files.

2. Root system files

README.md
Repository documentation

Explains how the project works, how to add content, and how to deploy it.

robots.txt
Search engine permission file

Tells search engines they can crawl your site.

sitemap.xml
Search engine page map

Lists important URLs for search engines.

LICENSE
Project license

Contains the MIT license text for your repository.

.gitignore
Git ignore rules

Tells Git which files or folders should not be committed.

3. CSS files

assets/css/theme.css
Theme variables

Controls colors, shadows, radii, fonts, and dark/light theme values.

assets/css/base.css
Base styles

Controls global page reset, typography, body background, and overall text behavior.

assets/css/layout.css
Layout system

Controls containers, grids, section spacing, and page-hero layouts.

assets/css/components.css
Reusable UI components

Controls cards, buttons, navbar, badges, detail page layout, code block styles, footer, and more.

assets/css/animations.css
Animation styles

Controls reveal effects, floating effects, and glow animation classes.

assets/css/responsive.css
Responsive design rules

Controls how the site adapts for phone and tablet screens.

4. JavaScript files

assets/js/theme.js
Theme logic

Handles dark/light toggle and remembers the chosen theme using localStorage.

assets/js/nav.js
Navigation logic

Handles sticky navbar behavior, mobile menu open/close, and active page links.

assets/js/animate.js
Animation engine

Shows reveal animations when sections enter the screen.

assets/js/loader.js
Data loader

Loads profile, skills, projects, and videos from JSON files.

assets/js/ui.js
UI renderer

Builds cards for projects, videos, and skills. Also stores reusable SVG icon markup.

assets/js/main.js
Page boot logic

Starts the normal pages, fills preview sections, and handles empty states.

assets/js/detail.js
Detail page engine

Loads a specific project or video using the URL id parameter and powers the copy-code system.

5. Data files

assets/data/profile.json
Creator profile data

Stores your name, tagline, links, about text, and current focus.

assets/data/skills.json
Skills data

Stores the skill cards shown on the website.

assets/data/index.json
Master content list

Lists all project JSON files and video JSON files. This is one of the most important files in the system.

Important: If you create a new project or video but do not add it here, it will not appear on the website.
assets/data/projects/project-XXX.json
Project data file

Each project has its own JSON file containing title, description, tags, repo URL, related video, and more.

assets/data/videos/video-XXX.json
Video data file

Each video has its own JSON file containing title, YouTube URL, related project, and source file list.

6. Code files

assets/code/video-001/main.py
Real tutorial source file

This is the actual source code shown on video detail pages. Users can copy it with one tap.

assets/code/video-XXX/...
Video code folder

Each video can have its own folder containing one or many files like main.py, helper.py, style.css, etc.

7. Image files

assets/images/favicon.svg
Browser icon

Appears in the browser tab and bookmarks.

assets/images/logo-mark.svg
Brand mark

Optional logo mark for identity and future design use.

assets/images/og-banner.png
Social sharing image

Used when the site is shared on social platforms.

assets/images/avatar.png
Profile image

Your avatar or branding image for profile use.

assets/images/projects/...
Project thumbnails

Images used by project cards and detail pages.
